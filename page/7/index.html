Netty网络框架

核心结论：Netty基于NIO多路复用和事件驱动模型，通过EventLoop处理IO事件，通过ChannelPipeline串联业务处理器，通过ByteBuf实现高效内存管理，是构建高性能网络服务的首选框架。

Netty通过EventLoop线程模型处理IO，通过ChannelPipeline串联处理器，通过ByteBuf管理内存，实现高并发低延迟网络通信

OSI七层模型定义网络通信的理论框架，TCP/IP四层模型是工程实现，二者核心层职责一致

OSI七层模型：从物理信号到应用数据的完整抽象

层级名称核心职责典型协议/设备数据单元
7应用层为用户提供网络服务接口HTTP、FTP、SMTP、DNS报文（Message）
6表示层数据格式转换、加密压缩SSL/TLS、JPEG、ASCII报文
5会话层建立、管理、终止会话NetBIOS、RPC报文
4传输层端到端可靠/不可靠传输TCP、UDP段（Segment）
3网络层寻址与路由选择IP、ICMP、路由器包（Packet）
2数据链路层局域网内节点间传输MAC、以太网、交换机帧（Frame）
1物理层比特流的物理传输光纤、双绞线、集线器比特（Bit）

PostgreSQL 架构原理与 MySQL 对比

核心结论：PostgreSQL 通过进程模型实现连接隔离、通过统一存储层管理数据、通过多版本并发控制（MVCC）实现无锁读写、通过丰富的索引类型和扩展体系覆盖从 OLTP 到 AI 向量的全场景，其设计哲学是"扩展优先、标准兼容、企业级可靠"；而 MySQL 以"简单优先、性能优先"著称，两者在架构模型、MVCC 实现、索引体系和扩展能力上存在根本性差异。

PostgreSQL 通过进程架构、统一存储层、MVCC 和扩展体系实现企业级可靠性

进程架构：每个连接独立进程，崩溃隔离优于线程模型

存储管理：表空间 + TOAST + 页面结构，大字段外置存储

MVCC 与事务：快照隔离 + 可见性规则 + Vacuum 清理

索引体系：B-Tree / GIN / GiST / BRIN 覆盖全场景

扩展生态：pgvector / PostGIS / Citus 等插件体系
