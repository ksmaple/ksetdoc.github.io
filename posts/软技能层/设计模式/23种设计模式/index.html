23种设计模式

核心结论：23种设计模式按创建型、结构型、行为型三大类组织，解决对象创建、类/对象组合、对象交互三类问题，掌握其适用场景比记忆实现细节更重要。

设计模式是面向对象设计的经验总结，在合适场景使用可提升代码复用性和可维护性

创建型模式（5种）：解耦对象创建

模式解决的问题典型应用
单例全局唯一实例Spring Bean默认单例、配置中心
工厂方法子类决定实例化哪个类JDBC DriverManager、日志框架
抽象工厂创建相关的产品族UI组件库（跨平台按钮/文本框）
建造者复杂对象的分步构建StringBuilder、Lombok @Builder
原型通过复制创建对象Java clone()、Spring原型作用域
// 建造者模式示例
Computer computer = new Computer.Builder()
    .cpu("Intel i9")
    .ram("32GB")
    .ssd("1TB")
    .build();

结构型模式（7种）：组合类/对象形成更大结构

模式解决的问题典型应用
适配器接口不兼容的转换Arrays.asList()、InputStreamReader
桥接抽象与实现分离JDBC驱动、不同数据库实现
组合统一处理单个和组合对象文件系统（文件+目录）、UI组件树
装饰器动态添加职责Java IO流（BufferedInputStream）
外观简化复杂子系统接口Spring的JdbcTemplate、工具类
享元共享细粒度对象字符串常量池、连接池、线程池
代理控制对象访问Spring AOP、MyBatis Mapper代理
// 装饰器模式：BufferedInputStream 包装 FileInputStream
InputStream in = new BufferedInputStream(new FileInputStream("file.txt"));

行为型模式（11种）：对象间职责分配与交互

模式解决的问题典型应用
责任链多对象依次处理请求Servlet Filter链、审批流程
命令请求参数化、可撤销Runnable、线程池任务
解释器定义文法解释器正则表达式、EL表达式
迭代器顺序访问聚合对象Java Iterator、Stream
中介者减少对象间直接交互MVC中的Controller、消息队列
备忘录捕获和恢复对象状态游戏存档、数据库事务回滚
观察者一对多依赖通知Spring Event、消息订阅
状态状态驱动行为变化订单状态机、线程状态
策略封装可互换算法Comparator、支付策略
模板方法算法骨架由子类实现AbstractList、JUnit测试框架
访问者分离算法与数据结构ASM字节码操作、编译器
// 策略模式示例
interface PaymentStrategy { void pay(double amount); }
class AlipayStrategy implements PaymentStrategy { ... }
class WechatStrategy implements PaymentStrategy { ... }

// 运行时切换策略
PaymentStrategy strategy = userPreferAlipay() ? new AlipayStrategy() : new WechatStrategy();
strategy.pay(100.0);

模式选型速查

场景推荐模式
需要全局唯一实例单例
对象创建逻辑复杂工厂/建造者
扩展功能不修改原类装饰器/代理
统一处理树形结构组合
运行时切换算法策略
解耦发送者和接收者观察者/命令
状态驱动不同行为状态
简化复杂系统调用外观

常见问题与误区

问题1：为使用模式而使用模式

根因：过度设计，简单问题复杂化 解决：KISS优先，模式是解决重复问题的工具，不是目标

问题2：模式混用导致代码难以理解

根因：一个类同时应用多个模式，职责混乱 解决：保持简单，一个类主要体现一个模式意图
