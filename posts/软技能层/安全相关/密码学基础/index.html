密码学基础

核心结论：密码学的三大基础工具是对称加密（快、共享密钥）、非对称加密（慢、公私钥分离）和哈希（不可逆、防篡改），实际系统中通常组合使用以兼顾安全与效率。

密码学通过数学难题保障信息的机密性、完整性和不可抵赖性

三大基础工具对比

类型特点密钥用途
对称加密加解密速度快共享同一密钥大量数据加密
非对称加密加解密速度慢公钥/私钥配对密钥交换、数字签名
哈希单向不可逆无密钥数据完整性校验、密码存储

对称加密

明文 + 密钥 → [加密算法] → 密文
密文 + 密钥 → [解密算法] → 明文

常见算法：AES、DES、3DES、SM4
算法密钥长度安全性说明
AES128/192/256位高目前主流标准
DES56位低已被暴力破解
3DES168位中DES的加强版，已逐步淘汰
SM4128位高国密标准

非对称加密

加密：明文 + 公钥 → 密文    解密：密文 + 私钥 → 明文
签名：明文 + 私钥 → 签名    验签：签名 + 公钥 → 验证

常见算法：RSA、ECC、SM2
算法密钥长度特点适用场景
RSA2048/4096位成熟广泛数字签名、HTTPS握手
ECC256位同等安全下密钥更短移动端、IoT资源受限场景
SM2256位国密标准国内合规要求

哈希函数

任意长度输入 → [哈希算法] → 固定长度输出（不可逆）

特性：
- 相同输入 → 相同输出
- 不同输入 → 输出几乎一定不同（碰撞概率极低）
- 无法从输出反推输入
算法输出长度安全性说明
SHA-256256位高目前主流
SHA-3可变高新一代标准
MD5128位低已发现碰撞，不推荐
SM3256位高国密标准

常见问题与误区

问题1：明文传输密钥

根因：对称加密的密钥在传输过程中被截获 解决：使用非对称加密交换对称密钥（如TLS握手），或采用安全的密钥分发机制

问题2：直接用哈希存储密码

根因：相同密码哈希值相同，容易被彩虹表攻击 解决：加盐哈希（Salt + Hash），如 bcrypt、scrypt、PBKDF2
