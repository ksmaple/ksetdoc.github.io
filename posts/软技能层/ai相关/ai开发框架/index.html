AI应用开发框架

核心结论：AI应用开发框架通过统一抽象封装不同大模型API的差异，提供流式响应、函数调用、RAG集成等能力，让开发者像使用普通SDK一样构建AI功能。

Spring AI核心概念

Prompt（提示词）
  ├── SystemMessage：设定角色和行为边界
  ├── UserMessage：用户输入
  └── 可选：Function调用声明

ChatClient（对话客户端）
  └── 统一接口，底层可切换 OpenAI/Claude/本地模型

VectorStore（向量存储）
  └── 接入ES、Redis、PgVector等存储Embedding

流式响应实现

// Server-Sent Events 流式输出
@GetMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<String> streamResponse(@RequestParam String question) {
    return chatClient.prompt()
        .user(question)
        .stream()
        .content();  // 逐字返回，前端实时展示
}
响应模式特点适用场景
同步阻塞简单直接，用户体验差后台任务
流式SSE逐字输出，体验好聊天对话
WebSocket双向通信多轮交互

Function Calling（工具调用）

// 定义可被AI调用的函数
public record WeatherRequest(String city, String date) {}

@Bean
public Function<WeatherRequest, String> weatherFunction() {
    return request -> {
        // 调用天气API获取数据
        return weatherService.query(request.city(), request.date());
    };
}

// AI根据用户问题决定是否需要调用
chatClient.prompt()
    .user("北京明天天气怎么样？")
    .functions("weatherFunction")
    .call();

常见问题与误区

问题1：流式响应与业务事务冲突

根因：流式输出期间数据库连接未释放 解决：在调用AI前完成所有数据查询，或调整连接池配置

问题2：Function Calling被滥用导致延迟

根因：AI频繁调用外部API，单次请求变多次 解决：合理设计Function粒度，设置超时和熔断，缓存常用数据
