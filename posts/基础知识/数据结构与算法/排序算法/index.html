排序算法

核心结论：排序算法的选择取决于数据规模、内存限制和稳定性需求，快速排序是通用场景首选，归并排序适合链表和外部排序，堆排序适合内存受限且需持续获取TopK的场景。

常用排序算法在时间复杂度、空间复杂度和稳定性三个维度各有侧重，没有绝对最优

快速排序：分治+原地分区的通用王者

public void quickSort(int[] arr, int left, int right) {
    if (left >= right) return;
    
    // 分区：选取基准，将数组分为 <pivot 和 >pivot 两部分
    int pivotIndex = partition(arr, left, right);
    
    // 递归排序左右两部分
    quickSort(arr, left, pivotIndex - 1);
    quickSort(arr, pivotIndex + 1, right);
}

private int partition(int[] arr, int left, int right) {
    int pivot = arr[right];  // 选最右为基准
    int i = left;
    
    for (int j = left; j < right; j++) {
        if (arr[j] < pivot) {
            swap(arr, i, j);
            i++;
        }
    }
    swap(arr, i, right);  // 基准放到正确位置
    return i;
}

快速排序的优化：

优化策略目的实现
三数取中法避免最坏情况（已排序数组）left, mid, right取中位数作为pivot
随机化概率避免最坏情况随机选取pivot
小数组切换插入排序减少递归开销长度<10时使用插入排序
尾递归优化减少栈深度先排序较短子数组
三路分区处理大量重复元素分为 <, =, > 三部分
// 三路快排（Dutch National Flag）
void quickSort3Way(int[] arr, int left, int right) {
    if (left >= right) return;
    
    int pivot = arr[left];
    int lt = left, gt = right, i = left + 1;
    
    while (i <= gt) {
        if (arr[i] < pivot) swap(arr, lt++, i++);
        else if (arr[i] > pivot) swap(arr, i, gt--);
        else i++;
    }
    // arr[left..lt-1] < pivot
    // arr[lt..gt] == pivot
    // arr[gt+1..right] > pivot
    
    quickSort3Way(arr, left, lt - 1);
    quickSort3Way(arr, gt + 1, right);
}

归并排序：稳定分治，链表和外部排序的首选

public void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;
    
    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid, right);
}

private void merge(int[] arr, int left, int mid, int right) {
    int[] temp = new int[right - left + 1];
    int i = left, j = mid + 1, k = 0;
    
    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) temp[k++] = arr[i++];
        else temp[k++] = arr[j++];
    }
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= right) temp[k++] = arr[j++];
    
    System.arraycopy(temp, 0, arr, left, temp.length);
}

归并排序的应用场景：

  • 链表排序：不需要额外空间（改变指针即可）
  • 外部排序：数据太大无法装入内存，分块排序后归并
  • 求逆序对：归并过程中统计跨越中点的逆序对数量

堆排序：原地排序 + TopK查询

public void heapSort(int[] arr) {
    // 建大顶堆（从最后一个非叶子节点开始下沉）
    for (int i = arr.length / 2 - 1; i >= 0; i--) {
        sink(arr, i, arr.length);
    }
    
    // 排序：将堆顶（最大值）放到末尾，调整堆
    for (int i = arr.length - 1; i > 0; i--) {
        swap(arr, 0, i);        // 堆顶放到已排序区
        sink(arr, 0, i);        // 对剩余元素重新建堆
    }
}

private void sink(int[] arr, int i, int n) {
    while (2 * i + 1 < n) {
        int j = 2 * i + 1;      // 左孩子
        if (j + 1 < n && arr[j + 1] > arr[j]) j++;  // 选较大孩子
        if (arr[i] >= arr[j]) break;  // 已满足堆性质
        swap(arr, i, j);
        i = j;
    }
}

实现原理与关键机制

Java中的排序实现（Dual-Pivot Quicksort）

Dual-Pivot快排 vs 传统快排：
- 选取两个基准（pivot1, pivot2），数组分为三部分
- 对小数组（<47）切换到插入排序
- 对大量重复元素切换到计数排序
- 实际性能比单基准快排高10-20%

TimSort：
- 归并排序 + 插入排序的混合
- 识别已排序片段（run），直接利用
- 稳定排序，最坏O(n log n)
- Python、Java、Android的默认排序算法

外部排序：内存不足时的排序策略

数据量：100GB，内存：1GB

1. 分块：将100GB分为100个1GB的块
2. 块内排序：每个块读入内存，用快排/归并排序
3. 多路归并：使用最小堆（优先队列），每次从100个块中
   取最小元素写入输出文件
   
时间：2 * 100GB（读一次写一次）磁盘IO
优化：增大块大小、增加归并路数（败者树）

实际应用中的决策与权衡

场景推荐算法原因
通用数组排序（Java）Dual-Pivot快排/TimSort标准库实现，高度优化
链表排序归并排序O(1)额外空间（改指针）
求TopK大元素最小堆（大小为K）O(n log K)，比全排序O(n log n)快
近乎有序数组插入排序O(n)最好情况
数据范围小（0-100）计数排序O(n+k)线性时间
稳定性要求归并排序/TimSort相等元素相对位置不变
内存极度受限堆排序O(1)额外空间

常见问题与误区

问题1：快速排序已经有序时性能退化

根因：每次都选最值作为pivot，分区极度不平衡 解决：三数取中法或随机化pivot选择

问题2：稳定排序一定比不稳定排序好？

根因：稳定排序保持相等元素相对顺序，有额外开销 结论：需要保持相对顺序时（如先按年龄排序再按分数排序）用稳定排序，否则不稳定排序更快

问题3：堆排序为什么实际比快排慢？

根因：堆排序缓存局部性差（跳跃访问），快排顺序访问缓存友好 结论：理论上都是O(n log n)，但常数因子差异导致快排实际更快
