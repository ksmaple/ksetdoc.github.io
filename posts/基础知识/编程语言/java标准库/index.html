// HashMap 关键源码：扰动函数减少哈希冲突
static final int hash(Object key) {
    int h;
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

HashMap扩容机制：

  • 默认初始容量16，负载因子0.75，触发扩容阈值12
  • 扩容为2倍，元素重新散列（JDK8后链表长度≥8转红黑树）
  • 预分配容量避免频繁扩容：new HashMap<>(预期数量 / 0.75f + 1)

// NIO 核心模式：Channel读取到Buffer，Selector监听多Channel
Selector selector = Selector.open();
ServerSocketChannel server = ServerSocketChannel.open();
server.configureBlocking(false);
server.register(selector, SelectionKey.OP_ACCEPT);

while (true) {
    selector.select();  // 阻塞等待就绪事件
    for (SelectionKey key : selector.selectedKeys()) {
        if (key.isAcceptable()) { /* 处理连接 */ }
        if (key.isReadable()) { /* 处理读取 */ }
    }
}

ArrayList vs LinkedList：随机访问与顺序访问的权衡

// ArrayList 随机访问：直接数组索引
public E get(int index) {
    rangeCheck(index);
    return elementData(index);  // O(1)
}

// LinkedList 随机访问：遍历链表
public E get(int index) {
    checkElementIndex(index);
    return node(index).item;  // O(n)，先判断index在前半还是后半
}

选型原则：

  • 随机访问多 → ArrayList
  • 头尾插入删除多 → LinkedList（或 ArrayDeque）
  • 二者都不绝对 → 默认 ArrayList（缓存局部性更好）

Stream vs 传统循环：不只是语法糖

// 传统循环：外部迭代
List<String> result = new ArrayList<>();
for (User u : users) {
    if (u.getAge() > 18) {
        result.add(u.getName().toUpperCase());
    }
}

// Stream：内部迭代 + 惰性求值 + 短路优化
List<String> result = users.stream()
    .filter(u -> u.getAge() > 18)
    .map(u -> u.getName().toUpperCase())
    .collect(Collectors.toList());

常见问题与误区

问题1：HashMap并发扩容导致死循环（JDK7）

根因：扩容时链表头插法导致环形链表 解决：升级到JDK8+（尾插法+红黑树），或使用ConcurrentHashMap

问题2：Arrays.asList()返回的List无法add/remove

根因：返回的是固定大小的内部类ArrayList，非java.util.ArrayList 解决：new ArrayList<>(Arrays.asList(arr)) 或 Java 9+ List.of()（同样不可变）

问题3：使用ExecutorService未关闭导致内存泄漏

根因：线程池中的工作线程是GC Root，持续持有引用 解决：try (ExecutorService pool = Executors.newFixedThreadPool(4)) { ... } 或手动shutdown()
