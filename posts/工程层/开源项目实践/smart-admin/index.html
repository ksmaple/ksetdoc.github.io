Smart-Admin管理系统

核心结论：Smart-Admin是一个基于Java + Vue3的中小型企业级中后台管理脚手架，通过模块化的组织架构（组织/部门/岗位/员工）和前后端分离设计，提供了开箱即用的SaaS多租户管理后台，适合作为内部系统或轻量级SaaS的启动模板。

Smart-Admin通过RBAC权限模型、多组织架构和标准化CRUD封装，降低中后台系统开发成本

技术栈

层级技术说明
前端Vue3 + Vite + Pinia + ElementPlus组合式API，开发体验好
后端Spring Boot + MyBatis-Plus主流Java技术栈
权限Sa-Token + RBAC注解式权限控制
多租户MyBatis-Plus TenantLine行级数据隔离
数据库MySQL支持多租户共享Schema

组织架构设计

租户（Tenant）
├── 组织（Organization）
│   ├── 部门（Department）
│   │   └── 员工（Employee）
│   └── 岗位（Position）
│       └── 员工（Employee）
└── 角色（Role）
    └── 菜单/按钮权限（Permission）
实体说明关系
租户SaaS顶层隔离单位所有数据的owner
组织企业/公司主体一个租户可有多个组织
部门树形结构支持层级汇报线
岗位职能标签与部门正交，可跨部门
员工系统用户关联部门+岗位+账号

权限模型

// 菜单权限（前端路由控制）
@SaCheckPermission("system:menu:query")
@GetMapping("/menu/list")
public List<MenuVO> list() { ... }

// 按钮权限（前端按钮显隐+后端接口校验）
@SaCheckPermission("system:user:add")
@PostMapping("/user/add")
public void add(@RequestBody UserAddForm form) { ... }
权限类型控制粒度实现方式
菜单权限页面级别前端路由守卫 + 菜单树过滤
按钮权限操作级别前端指令 + 后端注解校验
数据权限数据行级别注解拦截 + SQL动态拼接

标准CRUD封装

// Service层标准化
@Service
public class UserService extends ServiceImpl<UserMapper, User> {
    
    // MyBatis-Plus内置方法
    public IPage<User> page(UserQueryForm form) {
        LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
        wrapper.like(StringUtils.isNotBlank(form.getName()), User::getName, form.getName());
        wrapper.eq(form.getStatus() != null, User::getStatus, form.getStatus());
        return this.page(new Page<>(form.getPageNum(), form.getPageSize()), wrapper);
    }
}

前端标准化

// 通用表格Hook封装
export function useTable(api, params) {
    const loading = ref(false);
    const tableData = ref([]);
    const pagination = reactive({ pageNum: 1, pageSize: 10, total: 0 });
    
    const loadData = async () => {
        loading.value = true;
        const res = await api({ ...params, ...pagination });
        tableData.value = res.data.list;
        pagination.total = res.data.total;
        loading.value = false;
    };
    
    return { loading, tableData, pagination, loadData };
}

消息通知集成WebSocket或SSE推送

常见问题与误区

问题1：将脚手架直接用于超大规模SaaS

根因：未评估多租户方案是否满足未来数据量 解决：千万级以下用户行级隔离足够，更大规模需评估独立数据库方案

问题2：忽略数据权限导致横向越权

根因：只做了菜单权限，未做数据权限 解决：敏感接口必须加@DataScope，确保用户只能看到授权范围的数据
