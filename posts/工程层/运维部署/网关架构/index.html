网关架构

核心结论：生产环境的网关通常采用LBS云网关、Nginx分发网关、业务网关三层架构，每层负责不同粒度的流量管理，共同实现高可用、高性能、安全的流量入口。

三层网关架构分层处理流量，从外到内职责逐步聚焦

三层网关职责划分

用户请求
  ↓
LBS云网关（第一层）
  - 全局负载均衡（DNS/GSLB）
  - DDoS防护
  - SSL卸载
  - 边缘缓存（CDN）
  ↓
Nginx分发网关（第二层）
  - 七层负载均衡
  - 静态资源服务
  - 限流（连接数/速率）
  - 灰度发布
  ↓
业务网关（第三层）
  - 鉴权认证
  - 路由转发
  - API治理
  - 熔断降级
  ↓
业务服务集群
层级代表产品核心能力
LBS阿里云SLB、AWS ALB四层负载均衡、健康检查
NginxOpenResty、Nginx Plus七层路由、Lua扩展
业务网关Spring Cloud Gateway、KongAPI治理、微服务路由

Nginx核心配置

upstream backend {
    least_conn;                    # 最少连接算法
    server 10.0.1.10:8080 weight=5;
    server 10.0.1.11:8080 weight=5;
    server 10.0.1.12:8080 backup;  # 备用节点
}

server {
    listen 80;
    server_name api.example.com;
    
    location / {
        limit_req zone=api_limit burst=20 nodelay;  # 限流
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_connect_timeout 5s;
        proxy_read_timeout 10s;
    }
    
    location /static {
        alias /var/www/static;
        expires 30d;               # 静态缓存
    }
}

常见问题与误区

问题1：单点网关故障

根因：网关未集群部署 解决：网关多实例+Keepalived VIP、云厂商负载均衡

问题2：长连接导致负载不均

根因：Nginx默认轮询，长连接一直连在同一节点 解决：least_conn算法、设置连接超时
