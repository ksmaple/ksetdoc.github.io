活动系统

核心结论：活动系统通过配置化规则引擎支撑多变的运营需求，通过库存控制和发放流水保障奖品安全，通过防刷策略拦截恶意用户，核心挑战是灵活性与稳定性的平衡。

活动系统通过规则引擎实现灵活配置，通过库存和流水保障奖励安全，通过多层防刷策略保障公平

核心模块

模块职责
活动配置活动时间、参与条件、奖品配置
规则引擎判断用户是否满足参与条件
奖品发放库存扣减、发放记录、到账通知
防刷控制频次限制、设备/IP/账号维度风控

规则引擎设计

规则配置（JSON/DSL）：

{
  "rules": [
    { "type": "time", "start": "2026-01-01", "end": "2026-01-07" },
    { "type": "user_level", "min": 3 },
    { "type": "daily_limit", "count": 3 }
  ],
  "rewards": [
    { "type": "coupon", "id": "COUPON_001", "stock": 1000 },
    { "type": "points", "amount": 100 }
  ]
}

规则执行：
  1. 时间规则 → 当前时间在活动期内？
  2. 用户等级规则 → 用户等级 >= 3？
  3. 频次规则 → 今日参与次数 < 3？
  4. 全部通过 → 参与成功

奖品发放与防刷

发放流程：

1. 校验用户资格（规则引擎）
2. 扣减奖品库存（Redis DECR / DB乐观锁）
3. 记录发放流水（用户ID、奖品、时间）
4. 异步到账通知（MQ）

防刷策略：
- 账号维度：同一账号频次限制
- 设备维度：同一设备参与次数限制
- IP维度：同一IP参与次数限制
- 行为维度：异常行为模式识别（如秒级连续参与）

常见问题与误区

问题1：奖品超发

根因：并发扣减库存导致负数 解决：Redis原子扣减 + DB最终一致性

问题2：活动规则变更已生效活动

根因：配置未版本化 解决：规则版本控制，活动进行中禁止修改核心规则
