虚拟商城

核心结论：虚拟商城与实物电商的核心差异在于无需物流履约，库存为虚拟数字，重点在于商品定价策略、库存预扣与发放、以及虚拟道具的账户化管理。

虚拟商城通过虚拟库存管理和即时发放机制，实现商品购买到账户到账的闭环

核心模块

模块职责特点
商品管理虚拟商品上架、定价、分类无SKU，无物流属性
库存管理虚拟库存预扣、发放、回滚数字增减，无限可超卖
订单系统下单、支付、发货（到账）即时到账
账户系统虚拟资产（道具、皮肤、会员）账户内管理

库存管理设计

下单流程：

1. 用户下单 → 预扣库存
   UPDATE goods SET stock = stock - 1 WHERE id = ? AND stock > 0
   
2. 用户支付 → 确认扣减（已预扣，无需再扣）

3. 支付超时 → 回滚库存
   UPDATE goods SET stock = stock + 1 WHERE id = ?

虚拟商品特点：
- 可超卖（如限量商品可设置实际库存+候补）
- 发放即到账，无物流
- 支持组合包（Bundle）

常见问题与误区

问题1：高并发下库存超卖

根因：并发扣减库存导致负数 解决：数据库乐观锁（版本号）、Redis预扣、队列串行化

问题2：支付成功但道具未到账

根因：发放环节异常 解决：事务保证发放，失败重试，对账兜底
