统计系统

核心结论：统计系统通过埋点采集数据，通过实时链路（Flink/Kafka）提供秒级指标，通过离线链路（Hive/Spark）提供T+1精确报表，通过数据仓库分层实现数据治理。

数据采集：埋点设计

埋点类型说明示例
页面曝光用户进入页面page_view
点击事件用户点击按钮button_click
业务事件核心业务流程order_create, payment_success
性能事件接口耗时、错误api_latency, error_log
埋点数据格式：

{
  "event": "order_create",
  "timestamp": 1714400000000,
  "user_id": "10001",
  "device_id": "abc123",
  "platform": "iOS",
  "params": {
    "order_id": "O2023001",
    "amount": 199.00,
    "category": "electronics"
  }
}

实时 vs 离线计算

维度实时计算离线计算
延迟秒级~分钟级小时级~天级
准确性近似（可能丢数据）精确（全量数据）
技术栈Flink + KafkaSpark + Hive
用途实时监控、大屏财务报表、对账
存储Redis / DruidHive / ClickHouse

数据仓库分层

ODS（原始数据层）→ 埋点原始日志
  ↓ ETL清洗
DWD（明细数据层）→ 清洗后的业务明细
  ↓ 汇总
DWS（汇总数据层）→ 按维度汇总（日/周/月）
  ↓ 加工
ADS（应用数据层）→ 面向业务场景的报表数据

常见问题与误区

问题1：实时与离线数据不一致

根因：实时链路丢数据或计算逻辑差异 解决：以离线为准，实时供参考，建立差异监控

问题2：埋点数据量过大

根因：全量埋点导致存储和计算压力 解决：采样上报（1%）、分级埋点（核心/普通）
