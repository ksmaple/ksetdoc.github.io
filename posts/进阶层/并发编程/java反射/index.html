Java反射机制

核心结论：Java反射允许运行时检查和操作类结构，是框架实现解耦的基础，但性能开销较大；MethodHandle和VarHandle在JDK7+提供了更高效的替代方案，反射调用应尽量缓存和避免在热路径使用。

反射通过运行时元数据操作类成员，是框架基石但性能较差，MethodHandle提供了更高效的替代

反射基础：运行时窥探和操作类结构

// 获取Class对象
Class<?> clazz = Class.forName("com.example.User");
// 或
Class<?> clazz = User.class;
// 或
Class<?> clazz = user.getClass();

// 创建实例
User user = (User) clazz.getDeclaredConstructor().newInstance();

// 获取和调用方法
Method method = clazz.getDeclaredMethod("setName", String.class);
method.setAccessible(true);  // 绕过访问控制
method.invoke(user, "Tom");

// 获取和修改字段
Field field = clazz.getDeclaredField("age");
field.setAccessible(true);
field.setInt(user, 25);

反射的性能问题与优化

反射调用慢的原因：

1. 安全检查：每次调用都进行访问权限检查
   → 优化：setAccessible(true) + 缓存

2. 参数包装/解包：基本类型需装箱拆箱
   → 优化：使用特定类型方法（setInt, invokeExact）

3. 方法分派：无法内联优化
   → 优化：JIT编译后可能优化，但不稳定

4. 元数据查找：每次查找Method/Field对象
   → 优化：缓存Method/Field对象
// 反射优化：缓存Method对象
public class ReflectiveCache {
    private static final Method CACHED_METHOD;
    
    static {
        try {
            Method m = User.class.getDeclaredMethod("setName", String.class);
            m.setAccessible(true);  // 关闭安全检查
            CACHED_METHOD = m;
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
    
    public void setName(User user, String name) throws Exception {
        CACHED_METHOD.invoke(user, name);  // 复用缓存的Method
    }
}

MethodHandle：更高效的反射替代

// MethodHandle使用
MethodHandles.Lookup lookup = MethodHandles.lookup();
MethodType mt = MethodType.methodType(void.class, String.class);
MethodHandle mh = lookup.findVirtual(User.class, "setName", mt);

User user = new User();
mh.invoke(user, "Tom");  // 直接调用，无反射开销

// invokeExact要求参数类型精确匹配
mh.invokeExact(user, "Tom");  // 编译期检查类型

实际应用中的决策与权衡

场景方案原因
框架开发（Spring等）反射 + 缓存灵活但需要性能优化
高频热路径直接调用 / MethodHandle性能优先
Lambda表达式LambdaMetafactory编译器生成，性能最优
字段原子操作VarHandle（JDK9+）比反射Field高效
序列化/反序列化反射 + 缓存一次扫描，多次使用

常见问题与误区

问题1：反射调用私有方法失败

根因：未设置setAccessible(true) 解决：获取Method/Field后调用setAccessible(true)，配合缓存

问题2：反射性能差导致系统瓶颈

根因：热路径大量使用反射 解决：缓存Method/Field，或改用MethodHandle/代码生成

问题3：JDK9+模块系统限制反射访问

根因：JPMS强封装，默认不允许反射访问内部API 解决：添加–add-opens JVM参数，或升级库到兼容版本
