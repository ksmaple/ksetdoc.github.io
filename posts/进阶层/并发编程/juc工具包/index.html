JUC并发工具包

核心结论：JUC通过Lock体系替代synchronized提供灵活锁机制，通过Semaphore控制并发数，通过CountDownLatch和CyclicBarrier协调多线程执行节奏，通过CompletableFuture实现异步编排，覆盖了从互斥到协作的完整并发控制场景。

JUC通过显式锁、信号量、栅栏、异步Future四类工具，提供了比synchronized更精细的并发控制能力

Lock体系：比synchronized更灵活的锁机制

特性ReentrantLocksynchronized
可中断支持lockInterruptibly()不支持
可超时支持tryLock(timeout)不支持
公平性支持公平/非公平模式非公平
条件变量多个Condition一个wait/notify
尝试获取tryLock()无
性能JDK6+优化后接近同左
// ReentrantLock基本用法
ReentrantLock lock = new ReentrantLock();
lock.lock();  // 获取锁

try {
    // 临界区
} finally {
    lock.unlock();  // 必须在finally中释放
}

// 可中断锁
lock.lockInterruptibly();
try {
    // 可被Thread.interrupt()中断
} finally {
    lock.unlock();
}

// 尝试获取锁（不阻塞）
if (lock.tryLock(3, TimeUnit.SECONDS)) {
    try {
        // 获取到锁
    } finally {
        lock.unlock();
    }
} else {
    // 超时未获取到锁
}

ReadWriteLock：读写分离提升并发

ReadWriteLock rwLock = new ReentrantReadWriteLock();
Lock readLock = rwLock.readLock();
Lock writeLock = rwLock.writeLock();

// 读操作（多线程可同时读）
readLock.lock();
try {
    return cache.get(key);
} finally {
    readLock.unlock();
}

// 写操作（独占）
writeLock.lock();
try {
    cache.put(key, value);
} finally {
    writeLock.unlock();
}

// 锁降级：写锁可以降级为读锁（防止数据不一致）
writeLock.lock();
try {
    // 修改数据
    data = newData;
    readLock.lock();  // 获取读锁
} finally {
    writeLock.unlock();  // 释放写锁，保持读锁
}
try {
    // 使用数据（其他写线程被阻塞，读线程可并行）
    return data;
} finally {
    readLock.unlock();
}

Semaphore：限流控制并发数

// 最多允许10个线程同时执行
Semaphore semaphore = new Semaphore(10);

public void handleRequest() {
    try {
        semaphore.acquire();  // 获取许可，没有则阻塞
        // 执行业务逻辑
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    } finally {
        semaphore.release();  // 释放许可
    }
}

CountDownLatch vs CyclicBarrier：等待协作的不同模式

维度CountDownLatchCyclicBarrier
等待方一个或多个线程等待多个线程互相等待
计数器递减到0递减到0后重置
可重用否是
触发动作无可指定Runnable
典型场景等待N个任务完成分阶段并行计算
// CountDownLatch：主线程等待多个子线程完成
CountDownLatch latch = new CountDownLatch(3);

for (int i = 0; i < 3; i++) {
    final int index = i;
    new Thread(() -> {
        System.out.println("子任务" + index + "执行");
        latch.countDown();  // 计数减1
    }).start();
}

latch.await();  // 等待计数到0
System.out.println("所有子任务完成");

// CyclicBarrier：多线程分阶段协作
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("阶段完成，开始下一阶段");
});

for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        try {
            System.out.println(Thread.currentThread().getName() + "完成阶段1");
            barrier.await();  // 等待其他线程
            
            System.out.println(Thread.currentThread().getName() + "完成阶段2");
            barrier.await();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }).start();
}

CompletableFuture：异步任务编排

// 串行执行
CompletableFuture.supplyAsync(() -> fetchOrder(orderId))
    .thenApply(order -> calculatePrice(order))
    .thenCompose(price -> applyDiscountAsync(price))
    .thenAccept(finalPrice -> saveOrder(finalPrice))
    .exceptionally(ex -> {
        log.error("处理失败", ex);
        return null;
    });

// 并行执行后合并
CompletableFuture<String> userFuture = CompletableFuture.supplyAsync(() -> fetchUser(userId));
CompletableFuture<String> productFuture = CompletableFuture.supplyAsync(() -> fetchProduct(productId));

userFuture.thenCombine(productFuture, (user, product) -> {
    return "用户:" + user + ", 商品:" + product;
}).thenAccept(result -> System.out.println(result));

// 等待所有完成
CompletableFuture<Void> all = CompletableFuture.allOf(userFuture, productFuture);
all.thenRun(() -> {
    System.out.println("全部完成");
});

常见问题与误区

问题1：Lock未在finally中释放

根因：异常路径跳过unlock() 解决：始终使用try-finally模式

问题2：CountDownLatch计数不能重置

根因：设计为一次性使用 解决：需要重复使用用CyclicBarrier或重置新的CountDownLatch

问题3：CompletableFuture默认线程池问题

根因：默认使用ForkJoinPool.commonPool()，线程数=CPU核数-1 解决：指定自定义Executor，避免阻塞common pool
