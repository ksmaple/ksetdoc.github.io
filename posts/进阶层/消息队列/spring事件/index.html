Spring事件机制

核心结论：Spring事件机制通过发布-订阅模式实现应用内组件解耦，ApplicationEventPublisher发布事件，@EventListener接收事件，@Async实现异步处理，适合同应用内的轻量级事件通知。

Spring事件机制通过ApplicationEventPublisher和@EventListener实现应用内解耦，@Async支持异步消费

核心组件

组件说明
ApplicationEvent事件基类，自定义事件继承它
ApplicationEventPublisher事件发布接口
@EventListener标记方法为事件监听器
ApplicationEventMulticaster事件广播器，管理监听器

基本用法

// 定义事件
public class OrderCreatedEvent extends ApplicationEvent {
    private final Long orderId;
    private final Long userId;
    
    public OrderCreatedEvent(Object source, Long orderId, Long userId) {
        super(source);
        this.orderId = orderId;
        this.userId = userId;
    }
    // getter...
}

// 发布事件
@Service
public class OrderService {
    @Autowired
    private ApplicationEventPublisher publisher;
    
    public void createOrder(Order order) {
        // 保存订单...
        
        // 发布事件
        publisher.publishEvent(new OrderCreatedEvent(this, order.getId(), order.getUserId()));
    }
}

// 监听事件（同步）
@Component
public class OrderEventListener {
    
    @EventListener
    public void handleOrderCreated(OrderCreatedEvent event) {
        System.out.println("订单创建: " + event.getOrderId());
        // 发送短信、更新统计等
    }
    
    @EventListener
    public void handleOrderCreatedAsync(OrderCreatedEvent event) {
        // 监听器默认同步执行，会阻塞发布者
    }
}

异步事件处理

// 开启异步支持
@Configuration
@EnableAsync
public class AsyncConfig {
    @Bean(name = "eventAsyncExecutor")
    public Executor eventAsyncExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5);
        executor.setMaxPoolSize(10);
        executor.setQueueCapacity(100);
        executor.setThreadNamePrefix("event-");
        executor.initialize();
        return executor;
    }
}

// 异步监听
@Component
public class AsyncOrderEventListener {
    
    @Async("eventAsyncExecutor")
    @EventListener
    public void handleOrderCreated(OrderCreatedEvent event) {
        // 异步执行，不阻塞发布者
        sendNotification(event.getUserId());
    }
    
    @Async("eventAsyncExecutor")
    @EventListener
    public void handleOrderCreatedForStats(OrderCreatedEvent event) {
        // 同一事件可被多个监听器异步处理
        updateStatistics(event.getOrderId());
    }
}

事务事件

// 事务绑定的事件监听
@Component
public class TransactionalEventListener {
    
    // 事务提交成功后执行
    @TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
    public void afterCommit(OrderCreatedEvent event) {
        // 确保订单已持久化后再发送通知
        sendSms(event.getUserId());
    }
    
    // 事务回滚后执行
    @TransactionalEventListener(phase = TransactionPhase.AFTER_ROLLBACK)
    public void afterRollback(OrderCreatedEvent event) {
        // 补偿操作
        releaseStock(event.getOrderId());
    }
}

实际应用中的决策与权衡

场景方案原因
同应用内解耦Spring事件轻量、无需外部依赖
跨应用通信MQ（RocketMQ/Kafka）可靠、可扩展
事务后操作@TransactionalEventListener保证事务一致性
高并发事件异步监听器 + 线程池不阻塞主流程
顺序消费单线程监听器保持事件处理顺序

常见问题与误区

问题1：事件监听抛异常导致发布者失败

根因：同步监听器异常会传播到发布者 解决：监听器内部try-catch，或使用异步监听器

问题2：异步事件事务问题

根因：@Async后事务上下文丢失 解决：使用@TransactionalEventListener，在事务提交后触发

问题3：事件丢失

根因：应用重启，内存中的监听器消失 解决：重要事件使用持久化MQ
