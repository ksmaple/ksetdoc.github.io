MyBatis-Plus多租户

核心结论：MyBatis-Plus通过TenantLineInnerInterceptor实现SQL级多租户隔离，在ORM层自动拼接租户ID条件，相比独立数据库和共享数据库独立Schema方案，是成本和隔离性最均衡的SaaS数据隔离方案。

多租户方案对比

方案隔离级别成本复杂度适用场景
独立数据库物理隔离高低大客户定制、强合规要求

MyBatis-Plus行级隔离实现

@Configuration
public class MybatisPlusConfig {
    
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        interceptor.addInnerInterceptor(new TenantLineInnerInterceptor(new TenantLineHandler() {
            @Override
            public Expression getTenantId() {
                // 从当前线程上下文获取租户ID
                return new LongValue(TenantContextHolder.getTenantId());
            }
            
            @Override
            public String getTenantIdColumn() {
                return "tenant_id";  // 租户字段名
            }
            
            @Override
            public boolean ignoreTable(String tableName) {
                // 忽略不隔离的表（如全局配置表）
                return "sys_config".equals(tableName) || 
                       "sys_dict".equals(tableName);
            }
        }));
        return interceptor;
    }
}

-- 原始INSERT
INSERT INTO user (name, age) VALUES ('张三', 20);

-- 自动改写后
INSERT INTO user (name, age, tenant_id) VALUES ('张三', 20, 1001);

// 1. 登录时写入租户ID
public void login(LoginDTO dto) {
    User user = userService.validate(dto);
    TenantContextHolder.setTenantId(user.getTenantId());
    StpUtil.login(user.getId());
}

// 2. 拦截器统一设置
@Component
public class TenantInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest req, HttpServletResponse res, Object handler) {
        Long userId = StpUtil.getLoginIdAsLong();
        Long tenantId = userService.getTenantId(userId);
        TenantContextHolder.setTenantId(tenantId);
        return true;
    }
    
    @Override
    public void afterCompletion(...) {
        TenantContextHolder.clear();  // 清理防止线程复用污染
    }
}

// 3. ThreadLocal实现
public class TenantContextHolder {
    private static final ThreadLocal<Long> TENANT_ID = new ThreadLocal<>();
    
    public static void setTenantId(Long id) { TENANT_ID.set(id); }
    public static Long getTenantId() { return TENANT_ID.get(); }
    public static void clear() { TENANT_ID.remove(); }
}

    return "sys_config".equals(tableName);

    return isGlobalTable(tableName);
}

常见问题与误区

问题1：子查询或JOIN时租户条件遗漏

根因：复杂SQL中拦截器可能漏加租户条件 解决：检查SQL执行日志，手动补充wrapper条件，或使用@InterceptorIgnore在特殊方法中跳过

问题2：定时任务或异步线程租户ID丢失

根因：ThreadLocal无法跨线程传递 解决：异步执行前显式传递租户ID，或使用TransmittableThreadLocal
