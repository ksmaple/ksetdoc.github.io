MyBatis-Plus

核心结论：MyBatis-Plus在MyBatis基础上封装了通用CRUD、分页、代码生成等能力，通过SQL注入器和拦截器机制扩展功能，是提升MyBatis开发效率的增强工具而非替代方案。

MyBatis-Plus通过BaseMapper封装通用CRUD，通过SQL注入器和拦截器链扩展功能，减少80%的重复SQL编写

核心功能：从MyBatis到MyBatis-Plus的简化

MyBatis原始方式：

<!-- UserMapper.xml -->
<mapper namespace="com.example.mapper.UserMapper">
    <select id="selectById" resultType="User">
        SELECT * FROM user WHERE id = #{id}
    </select>
    
    <select id="selectList" resultType="User">
        SELECT * FROM user
        <where>
            <if test="name != null">AND name = #{name}</if>
            <if test="age != null">AND age = #{age}</if>
        </where>
    </select>
    
    <insert id="insert">
        INSERT INTO user (name, age, email) VALUES (#{name}, #{age}, #{email})
    </insert>
    
    <update id="updateById">
        UPDATE user 
        <set>
            <if test="name != null">name = #{name},</if>
            <if test="age != null">age = #{age},</if>
        </set>
        WHERE id = #{id}
    </update>
    
    <delete id="deleteById">
        DELETE FROM user WHERE id = #{id}
    </delete>
</mapper>

MyBatis-Plus方式：

@Mapper
public interface UserMapper extends BaseMapper<User> {
    // 无需编写XML，BaseMapper已提供通用CRUD
}

// 使用
User user = userMapper.selectById(1L);           // 根据ID查询
List<User> users = userMapper.selectList(null);   // 查询全部
userMapper.insert(user);                          // 插入
userMapper.updateById(user);                      // 根据ID更新
userMapper.deleteById(1L);                        // 根据ID删除

BaseMapper提供的通用方法：

方法说明
insert(T entity)插入
deleteById(Serializable id)根据ID删除
deleteByMap(Map<String, Object> columnMap)根据Map删除
delete(Wrapper wrapper)条件删除
updateById(T entity)根据ID更新
update(T entity, Wrapper wrapper)条件更新
selectById(Serializable id)根据ID查询
selectBatchIds(Collection<? extends Serializable> idList)批量查询
selectOne(Wrapper wrapper)查询单条
selectCount(Wrapper wrapper)查询总数
selectList(Wrapper wrapper)条件查询列表
selectPage(Page page, Wrapper wrapper)分页查询

条件构造器：Wrapper的链式查询

// QueryWrapper 条件构造
QueryWrapper<User> wrapper = new QueryWrapper<>();
wrapper.eq("status", 1)           // status = 1
       .like("name", "张")        // name LIKE '%张%'
       .between("age", 18, 30)    // age BETWEEN 18 AND 30
       .orderByDesc("create_time"); // ORDER BY create_time DESC

List<User> users = userMapper.selectList(wrapper);

// Lambda方式（类型安全）
LambdaQueryWrapper<User> lambdaWrapper = new LambdaQueryWrapper<>();
lambdaWrapper.eq(User::getStatus, 1)
             .like(User::getName, "张")
             .ge(User::getAge, 18);

// 链式查询（更简洁）
List<User> users = userMapper.selectList(
    Wrappers.<User>lambdaQuery()
        .eq(User::getStatus, 1)
        .like(User::getName, "张")
        .last("LIMIT 10")
);

实现原理与关键机制

SQL自动注入原理

MyBatis-Plus启动时扫描所有继承BaseMapper的接口：

1. 解析实体类（@TableName、@TableId、@TableField）
2. 为每个BaseMapper方法生成对应的MappedStatement
3. 注入到MyBatis的Configuration中

实体类映射：

@TableName("t_user")  // 表名
public class User {
    @TableId(type = IdType.AUTO)  // 主键，自增
    private Long id;
    
    @TableField("user_name")  // 字段映射
    private String name;
    
    @TableField(exist = false)  // 非数据库字段
    private String remark;
}

生成的SQL：
  SELECT id, user_name FROM t_user WHERE id = ?

分页插件：物理分页的实现

@Configuration
public class MybatisPlusConfig {
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        // 添加分页插件
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
        return interceptor;
    }
}

// 使用分页
Page<User> page = new Page<>(1, 10);  // 第1页，每页10条
Page<User> result = userMapper.selectPage(page, 
    Wrappers.<User>lambdaQuery().eq(User::getStatus, 1));

System.out.println("总记录数：" + result.getTotal());
System.out.println("总页数：" + result.getPages());
List<User> records = result.getRecords();

分页原理：

  • 拦截执行的SQL，包装为COUNT查询获取总数
  • 在原SQL后追加LIMIT子句
  • 将结果封装到Page对象

自动填充：创建时间和更新时间的自动维护

@Component
public class MyMetaObjectHandler implements MetaObjectHandler {
    
    @Override
    public void insertFill(MetaObject metaObject) {
        this.strictInsertFill(metaObject, "createTime", LocalDateTime.class, LocalDateTime.now());
        this.strictInsertFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
        this.strictInsertFill(metaObject, "deleted", Integer.class, 0);
    }
    
    @Override
    public void updateFill(MetaObject metaObject) {
        this.strictUpdateFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
    }
}

// 实体类配置
public class User {
    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createTime;
    
    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updateTime;
}

实际应用中的决策与权衡

场景推荐方案原因
简单CRUDBaseMapper + Wrapper零SQL编写
复杂查询自定义Mapper.xml复杂SQL用XML更直观
动态条件LambdaQueryWrapper类型安全，避免字段名硬编码
多表关联自定义SQL或MyBatis原生MyBatis-Plus不支持多表
逻辑删除@TableLogic自动处理deleted字段
乐观锁@Version自动处理版本号
批量操作saveBatch / updateBatchById批量插入优化

常见问题与误区

问题1：BaseMapper方法不够用怎么办

根因：MyBatis-Plus只提供单表通用操作 解决：自定义Mapper接口方法，配合XML或@Select注解

问题2：分页不生效或返回所有数据

根因：未配置PaginationInterceptor 解决：添加MybatisPlusInterceptor并注入PaginationInnerInterceptor

问题3：逻辑删除后还能查询到

根因：逻辑删除只是更新deleted字段，查询自动添加deleted=0条件 排查：确认@TableLogic配置正确，确认查询是否绕过Wrapper
